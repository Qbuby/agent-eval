# Agent-Eval E2E (Playwright)

End-to-end UI tests covering the evaluation workbench:

1. Login state (set up by `global-setup.ts`)
2. Evaluator CRUD (`/evaluators`)
3. Upload-source tab + preview (`/evaluation`)
4. **Real-agent run** — at least 1 non-error sample required
5. Detail page · `调用轨迹` project lookup
6. Multi-select → `/evaluation/compare` chart rendering

## Why this can't be faked

Tests 4 and 6 talk to a real agent. The contract enforced:

- run reaches a terminal status within 3 minutes
- `total ≥ 1`, **`error_count < total`** (at least one sample non-error)
- at least one sample has non-empty `actual_output`

If the agent dies mid-suite, those tests fail loud — the previous version
that passed against an unreachable URL was a false positive and has been
removed.

`global-setup.ts` probes the agent up-front and **refuses to run** if it
can't reach the agent. No "passed but didn't actually test anything" mode.

## Required environment

```bash
# Probe URL — visible to the host running playwright (use 127.0.0.1)
export AGENT_PROBE_URL=http://127.0.0.1:18094/api/agent/langgraph

# Agent URL — used inside the dockerized backend container
# host.docker.internal does not work on this WSL Docker setup;
# use the host's LAN IP instead. Find yours with `ipconfig`.
export AGENT_URL=http://172.16.100.188:18094/api/agent/langgraph
```

## Run

```bash
cd e2e
npm install
npx playwright install chromium
npx playwright test --reporter=list
```

HTML report at `playwright-report/index.html` after the run.

## Run mode

Headed by default (`headless: false` in `playwright.config.ts`). Pass
`PWDEBUG=1` for the inspector or `--headed=false` to switch to headless.

## Files

| Path | Purpose |
|---|---|
| `playwright.config.ts` | Test runner config — baseURL, timeouts, storageState |
| `global-setup.ts` | Probe agent + register/login a fresh user → `auth.json` |
| `tests/evaluation.spec.ts` | The 6 tests (serial mode) |
| `fixtures/cases.json` | 3 RPL201 questions, generated by the suite |

## Cleanup

Each run registers a fresh `e2e_<timestamp>` account and creates new
evaluator instances / case sources / runs in the database. Database
isn't reset between runs — old probe accounts and runs accumulate. Drop
the database volume (`docker compose down -v && docker compose up -d`)
if you want a clean slate.
